# 2.5.3 — 2026-05-30

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.9.13 → ^0.9.16 — enrichment block API, `ctx.enrich()` / `ctx.enrich.notice()`.
- **`clinicaltrials_search_studies`** — query/filter echo and empty-result recovery guidance now surface via `enrichment` (typed `enrichment` schema block + `enrichmentTrailer`) reaching both `structuredContent` JSON and the `content[]` markdown trailer. Previously these were output fields (`searchCriteria`, `noMatchHints`) on the `structuredContent` payload only; they have moved to the `enrichment` block.
- **`clinicaltrials_get_study_count`** — same enrichment adoption as `search_studies`: `searchCriteria` and `noMatchHints` output fields moved to `enrichment`; both channels now receive query echo and zero-count guidance.
- **`clinicaltrials_get_field_definitions`** — `searchQuery` output field moved to `enrichment`; empty-search `notice` added. Both channels receive the keyword echo and no-match suggestions when search mode returns nothing.
- **`clinicaltrials_find_eligible`** — `searchCriteria`, `funnel`, and `noMatchHints` output fields moved to `enrichment`. Search echo and funnel diagnostics now always populate the enrichment block; no-match `notice` (with funnel-stage diagnosis and actionable suggestions) fires when `studies` is empty.

## Dependencies

- `@cyanheads/mcp-ts-core` ^0.9.13 → ^0.9.16
