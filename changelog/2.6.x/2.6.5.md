# 2.6.5 — 2026-06-08

## Fixed

- **phaseFilter enum errors** now produce a param-named, actionable message with the valid values (`NA`, `EARLY_PHASE1`, `PHASE1`, `PHASE2`, `PHASE3`, `PHASE4`) instead of raw Essie parser text. The service catches `Allowed values for enum field … phases …` and maps it to `phaseFilter`. Resolves [#71](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/71).
- **sort format errors** now produce a param-named message with a format example (`FieldName:asc` / `FieldName:desc`) instead of opaque upstream text. Resolves [#71](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/71).
- **Dead code removed** — `upstreamParamInfo.phase` entry deleted; `phaseFilter` always routes through `filter.advanced` (Essie `AREA[Phase]`), so the API never emits `Invalid value in parameter \`phase\``. Resolves [#71](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/71).
- **Reserved-char docs corrected** on all 7 free-text query params across `search-studies.tool.ts`, `get-study-count.tool.ts`, `find-eligible.tool.ts`, and `recovery-hints.ts` — `,` acts as AND (does not fail); `( )` group sub-expressions and work when matched; only `[ ]` are reserved (used for `AREA[]` syntax). Resolves [#70](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/70).

## Changed

- `@types/node` 25.9.1 → 25.9.2 (dev).
