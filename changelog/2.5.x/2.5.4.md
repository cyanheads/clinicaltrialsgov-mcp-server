# 2.5.4 — 2026-05-30

## Fixed

- **`clinicaltrials_get_study_results`** — removed the unreachable `ids_not_found` error contract ([#55](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/55)). Since the batch-fallback (#32) converts all batch-level errors to per-ID `fetchErrors` entries, the declared error code was dead documentation that misled agents into expecting a thrown error when invalid NCT IDs are supplied.
- **`clinicaltrials_get_field_values`** — colloquial labels `RecruitmentStatus` and `RecruitingStatus` now auto-correct to the canonical `OverallStatus` (mirroring the existing case/whitespace normalization), so calls with those labels succeed rather than error ([#60](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/60)). The did-you-mean ranking now favors fields whose tokens the query covers more fully — `OverallStatus` surfaces above long niche fields that share only a common token.
