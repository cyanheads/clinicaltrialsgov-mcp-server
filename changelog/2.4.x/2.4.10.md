# 2.4.10 — 2026-05-20

## Fixed

- **`clinicaltrials_get_study_record` filter parity** ([#46](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/46)) — `locationLimit`, `outcomeLimit`, and `nearLocation` previously trimmed `format()` text while `structuredContent.study` kept the full upstream record, so the two surfaces drifted. Filters now apply once at the handler boundary and both surfaces render the same data. `nearLocation` annotates each surviving location with `distanceMi` in `structuredContent` too.

## Changed

- **`locationLimit` / `outcomeLimit` default to no cap.** Previously `10` and `5` with `0 = unlimited`. Now both are `.optional()` — omit for the full upstream list, pass an integer (`locationLimit` 1–500, `outcomeLimit` 1–100) to trim. Callers relying on the old silent trim need to pass an explicit cap.
- **Output schema replaces echo fields with `filtersApplied`.** The previous top-level `locationLimit`, `outcomeLimit`, `nearLocation` echoes are folded into a single `filtersApplied` object alongside new upstream totals (`totalLocations`, `locationsWithoutGeo`, `totalSecondaryOutcomes`, `totalOtherOutcomes`) so callers can detect what was dropped.
- **`clinicaltrials_search_studies` no longer fabricates `"Untitled"`** when `briefTitle` is missing — the title segment is omitted instead, matching the convention used elsewhere of preserving uncertainty rather than inventing labels.
- **Reserved-character notes tightened** on `clinicaltrials_search_studies.query` / `advancedFilter`, `clinicaltrials_get_study_count.query` / `advancedFilter`, and `clinicaltrials_find_eligible.conditions[]`. Phrasing dropped the Essie parser reference in favor of naming the offending characters directly (`` `[ ] ( ) ,` ``).
- **`clinicaltrials_find_eligible.location` description** now names the required `country` field and points to `clinicaltrials_search_studies.geoFilter` for radius searches across studies.
- **`clinicaltrials_get_field_definitions` description** lifts the canonical return shape (PascalCase identifier names) ahead of the usage-mode walkthrough so the value is visible before the mechanics.
- **`clinicaltrials_get_field_values` output `fields` description** spells out the per-field shape (path, piece name, type, missing/unique counts, top values, BOOLEAN special case).
