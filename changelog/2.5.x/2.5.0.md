# 2.5.0 — 2026-05-25

## Added

- **`clinicaltrials_get_field_definitions`** now requires an explicit `mode` field (`"search"` | `"drill"` | `"overview"`) instead of inferring mode from which optional params were present ([#49](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/49)). This was the dominant failure mode in production — the old mutually-exclusive optional design caused 10/10 field-definition errors. Each mode validates its required companion param (`query` for `search`, `path` for `drill`) and rejects unknown combinations.

## Fixed

- **`clinicaltrials_search_studies` / `clinicaltrials_get_study_count`** — upstream enum validation errors now name the tool's input param (`statusFilter`, `phaseFilter`) instead of the ClinicalTrials.gov API's internal param (`overallStatus`, `phase`), and include a recovery hint to call `clinicaltrials_get_field_values` with the relevant field ([#51](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/51)).
- **`clinicaltrials_find_eligible`** `noMatchHints` now identify the funnel collapse point and tailor suggestions accordingly: location-collapse hints lead with broadening the location; demographics/status-collapse hints lead with age/sex/status options; condition-collapse (or unknown) falls back to the generic set ([#50](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/50)).
- **`clinicaltrials_get_study_record`** description no longer references `structuredContent` transport framing — replaced with a note that `filtersApplied` preserves original totals when caps are applied ([#52](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/52)).

## Changed

- **`package.json` keywords** trimmed — removed `llm`, `llm-agent`, `clinicaltrials`, `api-client`, `hono`, `stdio`, `streamable-http`, `http`.
