# 2.8.1 — 2026-07-09

## Changed

- **Supply-chain install guard** — `bunfig.toml` adds `minimumReleaseAge` (holds npm publishes for 3 days before they're installable, excluding `@cyanheads/mcp-ts-core` for same-day framework adoption) and routes `bun install` through the Socket scanner (`@socketsecurity/bun-security-scanner`). `scripts/devcheck.ts`'s outdated-package check now recognizes rows bun marks as held by `minimumReleaseAge` as non-issues rather than false-positive failures.
- **`Dockerfile`** pins `oven/bun:1.3.14` in both stages (previously floating `1.3` / `1.3-slim`), adds `--ignore-scripts` to the build-stage install, and adds BuildKit cache mounts (`--mount=type=cache,target=/root/.bun/install/cache`) to all three install steps.
- **`prepare` script** now runs `git config core.hooksPath .husky` directly instead of invoking `husky` (see Dependencies).
- **`LICENSE`** copyright year → 2026.
- **`.gitattributes`** and **`.github/SECURITY.md`** added — line-ending/binary/generated-file rules and a vulnerability disclosure policy.

## Fixed

- **`clinicaltrials_search_studies` explicit-field long values** — `renderRequestedFieldsStudy` now lifts `maxValueLen` to `Infinity` (alongside the existing `maxLines`) when rendering user-requested fields, so a long primitive leaf (e.g. a 543-char `BriefSummary`) reaches `content[]` intact instead of clipped — matching `structuredContent`. ([#89](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/89))
- **`clinicaltrials_get_field_values` topValues truncation** — `content[]` now renders every fetched `topValues` entry instead of just the first 15; the disclosure line only appears when `uniqueValuesCount` exceeds the fetched count, and now names the real cause (the API's 250-value cap) instead of implying the tool itself trimmed the list. ([#90](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/90))
- **`clinicaltrials_find_eligible` per-study rendering and reproducible-query echo** — `format()` no longer caps interventions (was 3), locations (was 3, with a "+N more" tail), or central contacts (was 2), and no longer clips `briefSummary` at 200 characters; every field now matches what `structuredContent` already carries, and `formatRemainingStudyFields` lifts the same caps, mirroring #89. Separately, the `searchCriteria` enrichment (output schema, `ctx.enrich()`, and the `content[]` trailer) now carries the exact upstream `conditionQuery`, `statusFilter`, and `advancedFilter` strings plus `includeUnknownEnrollment=true`, so a caller can replay the full match set past the `maxResults` cap via `clinicaltrials_search_studies`. `docs/design.md` synced to the current output shape and corrected `sex` enum casing. ([#91](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/91))
