# 2.8.0 — 2026-06-30

## Changed

- **`search_studies` output-channel parity (breaking)** — `structuredContent.studies` now carries a compact per-study index by default: `nctId`, `briefTitle`, `overallStatus`, `phases`, `enrollmentCount`, `leadSponsor`, `conditions`, and a bounded `{ total, nearest }` locations summary — exactly what `format()` renders, not the full ~70KB study record. Callers needing specific leaves at full fidelity pass the `fields` parameter (each study then carries exactly the requested leaves, every location included); callers needing one complete record use `clinicaltrials_get_study_record`. Brings the primary search tool in line with the `content[]`/`structuredContent` parity invariant `get_study_record` and `get_study_results` already enforce. ([#86](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/86))
- **`@cyanheads/mcp-ts-core`** `^0.10.9 → ^0.10.10` — clears the transitive `js-yaml` advisory (see Security).
- **`@types/node`** `26.0.0 → 26.0.1` (dev).

## Fixed

- **`search_studies` explicit-`fields` location rendering** — a study fetched with explicit `fields` now renders every site in `content[]` via a dedicated loop. Previously the remaining-fields fallback deduped `locations[]` to the lead site (`SEARCH_RENDERED` suppresses the subtree), so sites `2..N` reached only `structuredContent`. ([#86](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/86))
- **`find_eligible` condition re-rank** — tier-2 "same concept or more specific subtype" credit now requires every significant token of a **multi-word** requested condition to be present in the study condition (word-order independent), replacing bidirectional substring containment. A single-word request no longer credits a distinct disease that merely contains it: `"Hypertension"` no longer lifts *Pulmonary Arterial Hypertension* to tier 2 (it falls to tier 1, below a genuine `"Hypertension"` exact match), while `"Type 2 Diabetes"` matches both `"Type 2 Diabetes Mellitus"` and `"Diabetes Mellitus, Type 2"`. `conditionMatchScore` is now exported. ([#79](https://github.com/cyanheads/clinicaltrialsgov-mcp-server/issues/79))

## Security

- **js-yaml DoS advisory cleared** — the `@cyanheads/mcp-ts-core ^0.10.10` lockfile re-resolve moved the transitive `js-yaml` dependency to the patched `3.15.0`, clearing [`GHSA-h67p-54hq-rp68`](https://github.com/advisories/GHSA-h67p-54hq-rp68) / `CVE-2026-53550` (moderate — quadratic-complexity DoS in YAML merge-key handling via repeated aliases). `bun audit` reports no vulnerabilities.
